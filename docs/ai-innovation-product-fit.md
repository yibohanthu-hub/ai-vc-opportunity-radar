# AI 创新产品岗位补充说明

## 作品是什么

**AI Opportunity Radar** 是我独立完成的可操作 Web Demo。它面向国内 VC 投资经理，把分散的 AI 创业项目和活动公开信息变成可筛选、可追溯、可解释、可收藏和可跟进的信息工作流。

线上作品入口：[`https://www.aivcradar.online/#/portfolio`](https://www.aivcradar.online/#/portfolio)
源码与自动化配置：[GitHub 仓库](https://github.com/yibohanthu-hub/ai-vc-opportunity-radar)

## 已实现的技术与产品闭环

- 使用真实公开来源构建**正式信息池**与**实时待核验队列**两层数据，而不是将新闻标题直接伪装为完整档案。
- 每个实体保留来源、更新时间、可信度、未知项和待验证问题；融资、估值、联系方式等无法证实的字段明确显示“未知 / 待核验”。
- 从 InfoQ 中文、量子位、36 氪、雷峰网公开 RSS / 融资快报和活动行 28 座重点产业城市公开目录实际采集候选，GitHub Actions 每 4 小时运行一次并静态发布；近 45 天候选会去重保留。
- 支持筛选、排序理由、详情核验、收藏、忽略、标签、跟进状态和批量整理；用户行为会记录为下一阶段可解释推荐的输入。
- 用浏览器实际操作验证了项目筛选、活动低价值识别、详情来源、收藏到跟进和实时队列回流；详细结果见 [`test-and-validation.md`](test-and-validation.md)。

## 评估设计与当前边界

本作品已完成的是**功能、数据与自动化发布验证**，并不虚构外部用户测试结论。下一阶段会用以下指标评价产品价值：

1. **发现有效线索率**：被用户点开、收藏或转为跟进的候选占比；
2. **核验效率**：从看到线索到定位关键来源、形成下一步问题的时间；
3. **信号质量**：人工判定为重复、过期、无关或字段失真的比例；
4. **行动转化**：收藏 → 待研究 → 待联系的转化率，以及用户对解释理由的采纳率。

这不是小鹏产品、车载系统、VLA 模型或机器人控制 Demo，也不声称已经解决这些问题。它证明的是我能够从 AI / 物理 AI 的技术信号、真实用户工作流、数据可信度和效果指标出发，独立把一个 0→1 的产品想法做成可访问、可操作、可验证的原型。
