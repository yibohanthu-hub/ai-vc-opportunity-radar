## 3. 异常处理与失败案例

### 重复候选

测试使用 `智谱 / 智谱AI`、`2026 WAIC / 世界人工智能大会`、`硅基流动产品新闻 / 公司官网` 三组候选。预期行为不是生成重复卡片，而是在名称标准化、来源链接或人工确认后合并为一个 canonical entity，并保留来源。

### 缺失融资和团队字段

面壁智能、星尘智能、帕西尼感知科技、思谋科技等公开入口在本次样本中无法提供足以确认的团队或融资事实。预期行为是：卡片仍进入信息池，但融资、估值和 B 轮以前可能性显示“未知 / 待核验”，可信度降低，并在详情页给出人工验证问题。

### 低价值活动

系统不直接删除中国国际数字经济博览会、自动化与 AI 科普大会和 AI Coding 分享活动。这三类活动仍可能有品牌、客户洞察或特定人脉价值；但它们没有显示出公开的项目发现机制或高密度交流结构，因此针对“发现项目”目标被标为低优先级。

### 修复记录

在浏览器测试中，批量选择使用整页重渲染会造成自动化点击短暂超时。已改为局部更新“已选择数量”，选择状态保持即时可用，随后批量操作再触发列表刷新。

## 4. 已知限制

1. 已核验样本不等于完整市场覆盖；实时队列是候选信息而非已验证投研结论。
2. 部分活动页面的最新嘉宾、时间或地点会变化，行动前必须再次打开官方来源。
3. 已实现轻量公开源采集和去重，但尚未实现跨来源实体消歧、字段级事实验证、组织账户、权限与 CRM 同步。
4. 浏览器本地存储只用于演示行为记录；刷新 / 更换浏览器后状态由当前 profile 决定。
