# AI Opportunity Radar · 产品说明

版本：V1.4 作品集 Demo｜国内优先样本快照：2026-07-17｜实时队列按运行时间更新

## 1. 用户与问题

目标用户是 VC 机构中负责 AI 项目发现、行业扫描与早期判断的投资经理。

他每天早晚都可能收到大量项目线索、新闻、社群信息、活动邀约和二手转述。原有工作流通常是：在搜索、社媒、活动页、聊天记录和表格之间切换；手动判断是否重复；把有限信息填入表格；再回到来源页核验。结果是“信息很多”，但很难快速回答：这家公司做什么、证据来自哪里、为什么现在值得再看一眼、下一步应该验证什么。

活动扫描也有类似问题。大型活动、论坛、闭门会、产品发布和颁奖典礼混在一起。活动的标题很吸引人，不代表对项目发现、创始人交流或产业判断有真实价值。

## 2. 产品定位

> 帮助 VC 投资经理持续发现国内未上市 AI 创业项目和高价值 AI 活动，并将公开信息转化为可筛选、可验证、可跟进的机会清单。

第一版优先覆盖国内公开信息；海外信号如需接入，只作为补充来源并在来源页明确标识，不主导首页、样本池或排序。

该产品不是自动投决系统。Agent 不输出“应该投资 / 不应该投资”；它负责发现、实体化、去重、初筛、解释和行动线索。投资经理保留判断、验证和触达决策。

## 3. 核心使用流程

```text
早 / 晚打开首页
  → 先看实时待核验队列和来源健康状态
  → 打开原始来源，判断是否值得进入正式信息池
  → 再看今日新增项目
  → 按赛道、地区、融资阶段线索、可信度筛选
  → 进入项目详情核对来源、理由与不确定性
  → 收藏 / 忽略 / 标签 / 设置跟进
  → 再看活动雷达，判断是否值得参会或回看
  → 在收藏与跟进页批量整理下一步行动
```

## 4. V1 功能范围

### 创业项目雷达

- 摘要卡片展示：项目名称、行业、产品描述、创始人、所在地、成立 / 公开出现时间、融资字段、B 轮以前可能性、来源、发现时间、更新时间和可信度。
- 支持按发现时间、行业、地区、融资阶段（含种子轮 / 天使轮 / Pre-A）、融资金额、B 轮以前可能性和可信度筛选；支持按综合信号、新颖性、来源可信度和跟进价值排序。
- 详情页提供完整产品信息、目标客户、AI 应用方式、团队、融资历史字段、商业化线索、竞争替代、公开来源、公开联系入口、Agent 关注理由、不确定性与下一步验证问题。
- 支持收藏、忽略、标签和跟进状态。

### AI 活动雷达

- 卡片展示时间、地点、形式、主题、活动类型、质量判断和项目发现潜力。
- 支持按时间、地点、主题关键词、活动类型、线上/线下、活动质量和项目发现可能性筛选。
- 详情页展示主办方、主题、嘉宾信息状态、目标人群、投资人价值、行动建议、来源和关联项目。
- 低价值不等于“坏活动”：它意味着不适合当前的“发现项目 / 深度交流”目标。例如邀请制品牌活动或颁奖典礼可能有品牌价值，但没有公开的项目发现机制。

### 实时待核验队列

- 真实读取国内公开 RSS、公开融资快报与公开活动目录（InfoQ 中文、量子位、36 氪融资快报、活动行北京 / 上海 / 深圳 / 杭州 / 广州 / 成都 AI 目录）；每条候选保留原始链接、抓取时间、来源状态、标题 / 摘要阶段线索和规则入队理由。
- 项目 / 产品新闻与活动不会直接变成正式投研档案；公司、融资、嘉宾、联系人等关键字段没有证据时均保留“未知 / 待核验”。
- 支持搜索、按候选类型和来源筛选、查看来源证据、收藏、忽略、标签和跟进状态。

### 早期项目发现边界

- 高校创业营、研究院孵化器、公司融资公告、投资方公告和产品上线页可作为早期项目的正式公开证据；详情页要标注成立 / 公开出现时间与证据类型。
- 小红书等公开内容社区只作为人工发现候选的入口：不登录、不绕过平台限制、不批量抓取，也不把帖子直接写成融资、团队或客户事实。
- 候选只有在公司、投资方、高校/孵化器或至少另一条可信公开来源交叉验证后，才进入正式项目池；否则留在实时待核验队列。

### 收藏与跟进

- 收藏、忽略、标签、跟进状态、最近查看时间和来源入口。
- 多选后批量设置跟进状态或标签。
- 保存查看、收藏、忽略、筛选、标签和跟进行为；所有行为仅存当前浏览器。

## 5. Agent 在做什么

| 环节 | Agent 产出 | 人的判断 |
| --- | --- | --- |
| 收集 | 国内 RSS、融资快报与公开活动目录中的候选；记录来源运行状态 | 决定哪些来源值得接入 |
| 规范化 | 名称、赛道、地点、融资字段、活动字段 | 检查字段口径 |
| 去重 | 将同一主体的多个公开来源合并到一个项目卡 | 处理边界模糊的公司主体 |
| 初筛 | 解释性信号分数、可信度、原因与风险 | 决定是否进一步研究 |
| 行动线索 | 验证问题、公开联系方式入口、相关活动 | 决定是否联系、如何联系 |
| 学习准备 | 收藏 / 忽略 / 标签 / 跟进等行为信号 | 决定哪些偏好真实有效 |

## 6. 为什么不只用普通搜索

搜索引擎擅长返回网页，但不天然解决投研工作中的实体管理和行动闭环。这个产品把分散的网页转成一组结构化对象：项目、团队、融资字段、活动、来源、证据、理由和下一步问题；并把用户行为保存为未来推荐的可解释信号。

普通搜索结果不会说明“为什么排在前面”，也不会把未知融资信息显式保留为未知。本产品要求每条重要信息关联公开来源和更新时间，并把不能确认的内容保留为待核验。

## 7. 验证与可信度原则

1. 每条项目和活动记录至少有一个公开来源链接。
2. 信息池写入时间与来源类型可见；来源的实际发布时间未知时，不伪造时间。
3. 融资阶段、融资额、估值、创始团队与联系方式不能从当前来源确认时，显示“未知 / 待核验”。
4. 可信度是来源覆盖与字段可确认性的提示，不是公司质量或投资价值评分。
5. 用户在行动前仍需打开原始来源，尤其是活动日期、嘉宾、公司主体和融资信息。

## 8. 个性化推荐的两阶段设计

V1 是信息池，不训练个性化模型。系统记录：查看、收藏、忽略、筛选条件、标签和是否进入跟进。第二阶段才可以给出能解释的主动推荐，例如：

> 你最近收藏了 3 个 Agent 基础设施项目，并将 2 个标为“待联系”，因此推荐这条 Agent 可观测性线索。

推荐必须展示相应行为信号，并允许用户关闭或修正偏好。

## 9. 当前限制

- 公开仓库已启用 GitHub Actions：每 6 小时采集一次并部署 GitHub Pages，且已完成一次端到端人工触发验证。GitHub Actions 的计划任务是尽力按时，不是金融级 SLA；页面以最近成功抓取时间和来源状态呈现新鲜度。
- 候选队列并不等于完整市场覆盖或事实核验；仍未实现跨来源实体消歧、字段级事实验证和人工审核升级。
- 没有登录、多人协作、组织权限、CRM、日历或邮件集成。
- 数据是有限的公开样本，而不是完整市场覆盖。
- 排序是显式规则样例，不是机器学习模型，也不构成投资建议。

## 10. 后续迭代

1. 增强每 6 小时国内公开源采集：补充字段级来源、抓取版本、来源变更告警与人工审核升级记录。
2. 引入可解释个性化推荐：从浏览、收藏、忽略、标签和跟进中学习，但保留用户控制。
3. 连接 CRM、日历与团队协作：只生成待审批任务和联系人入口，任何外部动作必须由人确认。
