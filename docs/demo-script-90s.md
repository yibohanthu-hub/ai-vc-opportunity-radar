# 90 秒 Demo 演示脚本

目标：在约 90 秒内展示“发现—整理—筛选—查看—收藏—跟进”的完整闭环。以下为建议口播，括号内为屏幕操作。

| 时间 | 操作 | 口播重点 |
| --- | --- | --- |
| 0–10 秒 | 打开首页 | “这是面向国内 VC 投资经理的 AI 机会雷达。它不是自动投决工具，而是把公开信号整理成可验证、可跟进的项目和活动清单。” |
| 10–22 秒 | 进入「实时待核验」，指向抓取时间、首次发现 / 最近见到和来源状态 | “这里是实际运行的国内公开信息入口：InfoQ 中文、量子位、36 氪公开 RSS / 融资快报，以及活动行 16 座重点产业城市目录会生成候选线索。每个来源都有本次抓取状态；近 45 天候选会去重保留，标题里的种子轮或天使轮只作为阶段线索，候选不会被伪装成已核实项目。” |
| 22–32 秒 | 打开第一条候选详情，查看来源、未知字段和核验问题 | “我先回到原始来源确认公司主体、融资或活动机制。系统明确告诉我为什么入队、还不知道什么，以及下一步该验证什么。” |
| 32–42 秒 | 收藏候选、设为「待研究」 | “收藏、标签和跟进会把真实候选纳入工作清单；是否进入正式信息池仍由投资经理决定。” |
| 42–54 秒 | 进入「创业项目」，筛选 `AI 基础设施`，打开 `硅基流动` | “正式项目池与实时候选分层。这里是国内优先的公开样本，融资和估值没找到可靠来源时就显示未知，不会补造。” |
| 54–66 秒 | 指向来源、Agent 理由、不确定性和验证问题 | “右侧不是‘建议投资’，而是为什么值得再读、当前不知道什么、以及下一步可以问什么。” |
| 66–76 秒 | 切换「AI 活动」，筛选 `低` 再清除，打开 `2026 世界人工智能大会（WAIC）` | “活动模块不仅收录高价值会议，也识别低优先级活动。低价值不代表不好，而是它缺少项目发现或深度交流机制。” |
| 76–88 秒 | 进入「收藏与跟进」，多选、批量设置状态 | “最后，项目、活动和实时候选都会在这里集中整理，显示下一步行动和公开来源。可以批量处理，不需要在多个表格之间跳转。” |
| 88–90 秒 | 停在跟进页 | “V1 已验证从公开信号到人工核验的闭环，并已按每 4 小时自动采集和发布；下一步是可解释推荐与团队协作。” |

演示前准备：清空浏览器本地存储或使用新的浏览器 profile，以保证收藏和状态从预期初始值开始。演示中最好收藏 `硅基流动`、添加“推理基础设施”标签，并在活动页打开 `2026 世界人工智能大会（WAIC）`。
