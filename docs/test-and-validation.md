## 3. 异常处理与失败案例

### 重复候选

测试使用 `Spine AI / YC company profile`、`Vapi / YC profile + official website`、`Broccoli / YC profile + company website` 三个候选。预期行为不是生成重复卡片，而是合并为一个 canonical project，并把多个来源保留在详情中。

### 缺失融资和团队字段

Manus、Genspark、Tasklet、Cartesia 等公开入口在本次样本中无法提供足以确认的团队或融资事实。预期行为是：卡片仍进入信息池，但融资、估值和 B 轮以前可能性显示“未知 / 待核验”，可信度降低，并在详情页给出人工验证问题。

### 低价值活动

系统不直接删除 AI House、MetaMorph 和 National AI Awards。这三类活动仍可能有品牌、客户洞察或特定人脉价值；但它们没有显示出公开的项目发现机制或高密度交流结构，因此针对“发现项目”目标被标为低优先级。

### 修复记录

在浏览器测试中，批量选择使用整页重渲染会造成自动化点击短暂超时。已改为局部更新“已选择数量”，选择状态保持即时可用，随后批量操作再触发列表刷新。

## 4. 已知限制

1. 样本不等于完整市场覆盖，也不代表实时数据。
2. 部分活动页面的最新嘉宾、时间或地点会变化，行动前必须再次打开官方来源。
3. 未实现真实的跨源爬取、新闻增量、组织账户、权限与 CRM 同步。
4. 浏览器本地存储只用于演示行为记录；刷新 / 更换浏览器后状态由当前 profile 决定。
